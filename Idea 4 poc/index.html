Project Orion - Backend Authentication PRD
1. System Overview
This document outlines the architecture for the new Orion dashboard backend services. The services will expose GraphQL endpoints for the frontend.
2. Data Storage & Lifecycle
User profile data and session logs will be stored in Amazon RDS. To optimize storage costs, all authentication audit logs will be retained for a maximum of 30 days before being automatically purged. User passwords will be hashed using SHA-256.
3. Service Architecture
The authentication microservice will be written in Node.js and deployed as an independent container cluster out of the US-East-1 region. We expect to maintain a 99.5% uptime SLA for this specific component.
