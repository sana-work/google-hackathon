DOCUMENT TYPE: Vendor Contract Extract
CONTRACT: Cloud Infrastructure and Storage Services Agreement
VENDOR: CloudVault Storage Solutions Pvt. Ltd.
CONTRACT ID: VND-2024-0312
EFFECTIVE DATE: 2024-04-01
EXPIRY DATE: 2027-03-31
ANNUAL CONTRACT VALUE: ₹3.2 Crore
RELATIONSHIP: CRITICAL Vendor — Primary cloud storage provider

═══════════════════════════════════════════
SECTION 1 — SERVICES PROVIDED
═══════════════════════════════════════════

1.1 CloudVault provides managed cloud storage, object storage, and database hosting services for the organization's non-payment production workloads.

1.2 Services include: block storage, object storage (S3-compatible), managed PostgreSQL, managed MongoDB, backup and snapshot management, and disaster recovery infrastructure.

═══════════════════════════════════════════
SECTION 2 — DATA CENTER LOCATIONS
═══════════════════════════════════════════

2.1 Primary Data Center: Mumbai, India (IN-West-1)
2.2 Disaster Recovery Data Center: Singapore (SG-Central-1)
2.3 Backup Storage: Singapore (SG-Central-1) — daily incremental backups replicated from Mumbai

2.4 Data Sovereignty Clause: CloudVault SHALL NOT move, replicate, or process any organization data outside of the Mumbai and Singapore regions without prior written consent.

2.5 EU Data: The organization has NOT contracted CloudVault for EU data storage. Any EU personal data currently flowing to CloudVault infrastructure (e.g., via CRM DR replication) is outside the contractual scope and must be remediated.

═══════════════════════════════════════════
SECTION 3 — SECURITY AND COMPLIANCE
═══════════════════════════════════════════

3.1 Certifications: CloudVault maintains SOC2 Type II, ISO 27001:2022, and ISO 27017 certifications.

3.2 Encryption: CloudVault provides encryption at rest using AES-256 with Google-managed keys (GMEK). Customer-managed encryption keys (CMEK) are available as an add-on service but are NOT currently included in this contract.

3.3 Access Controls: CloudVault employees access customer environments only for break-glass support with customer approval. All access is logged and auditable.

═══════════════════════════════════════════
SECTION 4 — BREACH NOTIFICATION
═══════════════════════════════════════════

4.1 CloudVault will notify the organization of any confirmed data breach within 72 hours of detection.
4.2 CloudVault will provide a detailed incident report within 14 days of breach notification.
4.3 CloudVault will cooperate with the organization's incident response team and regulatory authorities as required.

═══════════════════════════════════════════
SECTION 5 — SUB-PROCESSORS
═══════════════════════════════════════════

5.1 CloudVault currently uses the following sub-processors:
    (a) NetBackup India — backup management services (India)
    (b) SG DataCenter Ops — physical facility management (Singapore)

5.2 CloudVault will notify the organization 30 days before engaging any new sub-processor.

═══════════════════════════════════════════
SECTION 6 — TERMINATION AND DATA RETURN
═══════════════════════════════════════════

6.1 Upon termination, CloudVault will return all data in a portable format (JSON/CSV + database dumps) within 45 days.
6.2 CloudVault will certify complete data deletion within 90 days of data return.
6.3 Post-termination audit rights: 12 months.
